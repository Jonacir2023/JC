# 📌 Architecture Decision Records (ADRs)

Log centralizado de decisões importantes do projeto. Cada decisão deve ser documentada com contexto, alternativas consideradas e consequências.

## Decisões Registradas

*Nenhuma ADR registrada ainda. Use o template abaixo para adicionar.*

## Template ADR

```markdown
# ADR-[NÚMERO]: [Título da Decisão]

## Status
[Proposto | Aceito | Descartado | Depreciado]

## Contexto
Qual era o problema ou pergunta que motivou esta decisão?

## Decisão
Qual decisão foi tomada?

## Alternativas Consideradas
- Opção A — Razão pela qual foi rejeitada
- Opção B — Razão pela qual foi rejeitada
- Opção C — Por que foi escolhida

## Consequências
- Positivas:
- Negativas:
- Neutras:

## Data
[YYYY-MM-DD]

## Refs
- [[02-Architecture/README|Architecture]]
- Link para issue/PR
```

---

**Dica**: Mantenha ADRs curtas e focadas. Uma decisão = um arquivo.
